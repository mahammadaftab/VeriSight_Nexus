PART 2: DEEP DIVE: THE 8-AGENT INVESTIGATION PIPELINE (Minutes 3-10)
--------------------------------------------------------------------------------
[What you say when they ask: "Explain your agent architecture. How does data flow?"]

"Data flows strictly and sequentially. You can see this logic inside `code/api.py`.
1. Claim Extraction Agent (`agents/claim_extraction_agent.py`): It reads the user chat and definitively extracts what object and part they are complaining about.
2. Vision Inspection Agent (`agents/vision_inspection_agent.py`): The multi-modal expert. It strictly analyzes the image pixels. It flags blurry images and identifies physical damage.
3. Object Part Alignment Agent (`agents/object_part_agent.py`): *This is my anti-hallucination layer.* It is a programmatic referee. If the user chat claims a 'cracked windshield', but the Vision agent sees a 'dented door', this agent flags a hard contradiction.
4. Risk Assessment Agent (`agents/risk_assessment_agent.py`): It analyzes `user_history.csv` to flag behavioral fraud patterns.
5. Evidence Validation Agent (`agents/evidence_validation_agent.py`): It ensures the images meet the strict `evidence_requirements.csv` rulebook.
6. Decision Engine (`agents/decision_agent.py`): A rules-based aggregator that combines the 5 reports above into the final verdict ('supported', 'contradicted', or 'not_enough_information').
7. Explanation Agent (`agents/explanation_agent.py`): Translates the verdict into a human-readable justification.
8. Investigation Copilot (`agents/copilot_agent.py`): Generates a natural language brief specifically for the frontend UI."

--------------------------------------------------------------------------------
PART 3: THE TECHNICAL & ENGINEERING FLEX (Minutes 10-18)
--------------------------------------------------------------------------------
[What you say when they ask: "LLMs are unpredictable. How did you guarantee the output matched our strict 14-column CSV format?"]

"I completely eliminated output unpredictability by using Pydantic Structured Outputs combined with OpenRouter JSON Mode (`code/services/llm_service.py`). I dynamically inject a Pydantic `model_json_schema()` directly into the prompt. When the LLM responds, I validate it using `model_validate_json()`. This mathematically forces the LLM to adhere to my strict Enums. It cannot hallucinate column names."

[What you say when they ask: "How did you handle LLM API Rate Limits and Latency?"]

"I engineered three enterprise-grade reliability optimizations:
1. Exponential Backoff: In `llm_service.py`, I used the Python `tenacity` library. If an LLM call fails due to a 429 Rate Limit, it automatically retries with an exponential delay up to 7 times.
2. MD5 Image Caching: I built a `CacheService` (`code/services/cache_service.py`). It hashes incoming images and prompts. If a user uploads the same image, my system retrieves the JSON from `.cache/` in 2 milliseconds, bypassing the LLM completely.
3. Asynchronous Batching: In `generate_all.py`, I used Python's `asyncio.gather` to concurrently process massive batches of claims, dropping processing time drastically."

--------------------------------------------------------------------------------
PART 4: THE FULL-STACK ARCHITECTURE (Minutes 18-24)
--------------------------------------------------------------------------------
[What you say when they ask: "Tell me about your tech stack and UI."]

"The backend is a stateless Python FastAPI server. It handles the heavy AI orchestration and writes final audit logs to a MongoDB database (`database.py`). 

The frontend is a cinematic Next.js App Router application written in TypeScript and styled with Tailwind CSS. I didn't want the AI to feel like a black box, so I implemented FastAPI WebSockets (`@app.websocket("/ws/analyze/{user_id}")`). This allows the frontend dashboard to render a live 'Execution Pipeline'. A fraud analyst can literally sit at the dashboard and watch the 8 agents pass data to each other in real-time."

Q1: "Why use OpenRouter instead of just hardcoding the Gemini API?"
Your Answer: "OpenRouter makes the system vendor-agnostic. By using the standard OpenAI-compatible SDK in `llm_service.py`, I can hotswap the intelligence engine from Gemini 2.5 to Claude 3.5 Sonnet to GPT-4o with a single string change in the `.env` file. No code rewrites required."

Q2: "What happens if a user submits a picture of a completely dark room?"
Your Answer: "The system catches it instantly. The Vision Inspection Agent will set `is_valid_image` to False. The Evidence Validation Agent will see that flag, immediately fail the claim, and the Decision Engine will output 'not_enough_information'. It never blindly guesses."

Q3: "If you had 1 more week to work on this, what would you add?"
Your Answer: "I would replace my local file-based MD5 Cache with Redis for distributed, multi-server caching. I would also add Dockerfiles to deploy the FastAPI and Next.js layers into a Kubernetes cluster."
